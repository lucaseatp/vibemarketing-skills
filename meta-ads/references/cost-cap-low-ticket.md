## When to Use

For products with ticket **below ~$20 USD equivalent** (adjust for local currency):

- **Do not** apply aggressive daily scaling on the same campaign — low ticket caps ROAS per extra sale.
- Prefer **stacking** multiple 1-1-1 campaigns with fixed budget + **one cost cap / target cost** campaign for volume.

Works best when **account has data**: validated creatives, known historical CPA.

## Static Budget (1-1-1)

Base setup:

| Parameter | Reference |
| --- | --- |
| Structure | 1-1-1 per validated creative |
| Daily budget | **~$8–12/day** (adjust to currency/ticket) |
| Logic | 1 sale ≈ breakeven; 2–3 sales ≈ profit; higher ticket (upsell) improves margin |
| Daily scaling | **Do not** raise budget just because it sold well today |
| Bad day (~$7 spent, no sale) | Pause or **reduce** budget → redirect to what's selling |
| Good day (1–3 sales) | Keep static budget; volume comes from **cost cap** |

Goal: **many** stable 1-1-1 campaigns, not one giant campaign.

## Cost Cap / Target Cost Campaign

Second scale layer — **concentrated volume**:

1. Identify **top 3–5 ads** already isolated and profitable (1-1-1).
2. Create new campaign with **sales/conversion** objective.
3. Bid strategy: **cost cap / target cost per result** (not pure "maximize conversions" for this playbook).
4. Place **5 validated** creatives in same ad set (1-3 or 1-5 ads).
5. Starting budget: **$200+/day** (scale with performance).
6. **Do not** turn off isolated 1-1-1s — same creative can run in both structures with different bids.

Expected behavior:

- Days 1–2: may spend little (learning).
- Days 3–4+: tends to "pick up pressure" and drive **volume** while keeping CPA near target.

Real example (ebook, ticket $9–13, target CPA $7): cost cap campaign doing 22–36 sales/day with profit > $200/day, while 1-1-1s stay at ~$10/day static.

## How to Set Target CPA

1. Filter **profitable** campaigns in period (e.g. current month, positive ROAS).
2. Calculate **average CPA** of those campaigns.
3. Set cost target **slightly above** winning average CPA (e.g. avg CPA $3 → target $4–5).

Reason: compete in auction **without** overpaying, using real history — not guesses.

Include **order bumps** in margin math, not base price only.

## Daily Operations

| Signal | Action |
| --- | --- |
| Good ROAS/CPA today, 1-1-1 campaigns | Keep ~$8–12/day; no daily scaling |
| Extra volume needed | Raise budget on **cost cap** campaign |
| Static campaign no sale today | Reduce (e.g. ~$6) or pause |
| Creative ROAS dropped but cheap IC | Can keep low budget (assist role) |
| Creative losing 7–14 days, only one in ecosystem | Pause and **launch replacements** urgently |

## Duplicate Creative Across Structures

- Creative in **isolated 1-1-1** + same creative in **cost cap** = ok.
- **Different bid strategies**, not blind cannibalization.

Cost cap **does not** replace test pipeline — continue 1-1-3 for new creatives.

## Limits

- Cost cap without **historical CPA volume** → guessed target, campaign won't spend or spends poorly.
- Very low ticket + tight CPA target → zero delivery.
- Don't count WhatsApp/engagement as sale if pixel doesn't track — cost cap needs reliable **purchase/lead** event.
